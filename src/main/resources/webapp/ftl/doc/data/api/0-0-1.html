## 1. 传输方式 #### 为保证交易安全性，采用HTTPS传输 ## 2. 提交方式 #### 采用POST方法提交 ## 3. 数据格式 **请求和响应包体格式统一为JSON** ### 3.1 请求 **请求参数分两块，一块是公共参数，一块是业务参数** #### 3.1.1 公共参数URL传递，如下： ```text https://openapi.8.163.com/xxx?timestamp=2017-01-01 08:08:08&app_id=59B0294BC36040899E12E0EA137D4939&sign_type=RSA2&version=1.0 ``` #### 3.1.2 业务参数和签名统一放body中，结构如下： ```json { "sign": "签名，签名算法见 5.1", "data": "业务参数，BASE64" } ``` **业务参数目前仅支持JSON，也就是说：这里的data数据是把JSON字符串BASE64之后的数据** ### 3.2 响应 #### 响应body结构 ```json { "sign": "签名，验签算法见 5.2", "data": "业务数据，BASE64", "code": "网关返回码", "msg": "网关返回码描述" } ``` **业务数据，也是JSON结构，其BASE64解码出来格式如下：** ```json { "code":"业务返回码", "msg":"业务返回码描述", "example1":"业务字段，具体API定制", "example2":"业务字段，具体API定制" } ``` * [戳查看网关返回码](http://note.youdao.com/noteshare?id=2ce3e300d7d88c6b8d51bab14dd8acec&sub=8F9ADCF041374E079F6854883073BA3E) ## 4. 字符编码 #### 统一采用UTF-8字符编码 ## 5. 签名算法 **签名算法目前分两类：** * 一类针对后端，目前仅支持SHA256WithRSA，[戳查看算法详情](ppp) * 一类针对APP端，目前仅支持SHA256WithAES，[戳查看算法详情](http://note.youdao.com/noteshare?id=fca0a5ae5d090dd8ce83b695eb65f994&sub=9D8F21FEF7A04B9786A4CE151E614EBA) * ## 6. 判断逻辑 #### 先判断网关返回码，再判断业务返回码
